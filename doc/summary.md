## 数据评估与标准预测方法

### 当前数据概览

探索结果显示项目已有相当丰富的数据：

**数据强项 ✅**
- **Pre-match xG** - 最重要的预测特征之一
- **Pinnacle 赔率** - 业界公认最有效的"聪明钱"市场信号
- **主客场分拆统计** - xG、PPG、胜率全部有 home/away breakdown
- **近期状态** (last 5 matches) + 连胜/连败 streak
- **H2H 历史** - 9 个字段的交锋记录
- **全面投注市场** - BTTS、过2.5、角球等衍生市场
- **赛季累积统计** - 50+ 字段，含净胜球、失分率、BTTS%

---

### 关键缺口 ❌

| 缺失数据 | 重要性 | 说明 |
|---------|--------|------|
| **伤停/上场名单** | 极高 | 只有 top scorers，缺少关键球员缺阵信息 |
| **ELO/动态强度评分** | 高 | FootyStats 不提供，需自己用历史数据计算 |
| **多赛季历史比赛数据** | 高 | 训练 ML 模型至少需要 2-3 赛季逐场数据 |
| **赛程密度/疲劳** | 中 | 近 7 天比赛数量、欧战/联赛双线 |
| **裁判统计** | 中 | 只有 referee_id，需解析为判罚风格特征 |
| **转会窗期阵容变化** | 中 | 赛季中买卖球员会导致统计数据失真 |

### 足球预测的标准方法

学术界和量化玩家公认的方法分两大类：

#### 1. 统计模型（最成熟）

```
Dixon-Coles 模型 (1997，经典)
├── 用泊松分布建模主客队进球数
├── 引入低比分修正参数 (0-0, 1-0, 0-1, 1-1 发生率异常)
├── 时间衰减权重（近期比赛权重更高）
└── 输出：P(主胜) / P(平) / P(客胜)
```

#### 2. 机器学习（近年主流）

```
特征工程 → Gradient Boosting (XGBoost/LightGBM)
├── 滚动窗口特征: 近5/10/38场 xG、得分、积分
├── ELO 强度差
├── 主客场调整系数
├── 赔率隐含概率 (closing odds)
└── 输出: 三分类概率
```

#### 标准流水线

```
1. 数据收集
   历史比赛 (2-3赛季) + 实时赔率 + 阵容

2. 特征工程
   ├── 滚动 xG 均值 (指数加权)
   ├── ELO 差值
   ├── 主客场 PPG
   ├── H2H 历史
   └── 赔率隐含概率 (去除水钱后)

3. 模型训练/验证
   时序交叉验证 (walk-forward，不能随机切分！)

4. 概率校准
   Platt Scaling 或 Isotonic Regression

5. 找 Edge（关键步骤）
   模型概率 vs Pinnacle 隐含概率 → 正期望值时下注
   EV = p_model × odds - 1

6. 仓位管理
   Kelly Criterion 或分数 Kelly
```

### 当前架构的核心问题

项目现在用 **LLM 做分析**，这是原型阶段合理的选择，但有明显局限：

1. **无法训练/回测** - LLM 不能在历史数据上系统验证准确率
2. **一致性差** - 相同输入 LLM 可能给出不同判断
3. **缺乏校准** - LLM 输出的"70% 信心"没有统计意义

**建议的演进路径：**

```
当前: LLM 定性分析
  ↓
短期: LLM + 统计模型双轨，互相验证
  ↓
中期: Dixon-Coles 或 xG-Poisson 作为基础模型，LLM 处理软因素（伤停、动机）
  ↓
长期: 历史数据训练 XGBoost，LLM 作为异常检测层
```

### 最急需补充的数据

如果只能加一个数据源，优先级：

1. **伤停/首发名单** (Sofascore / API-Football 有这个) — 对赔率影响最大
2. **多赛季历史数据** — 用 FootyStats league-matches 批量拉取历史比赛，建本地数据库
3. **Closing Pinnacle odds** — 已有，是最强的预测信号，确保在比赛开始前记录

总体来说，FootyStats 的数据对于 LLM 分析阶段是**足够的**，但要做严肃的量化预测，还需要补充历史数据和伤停信息，并建立可回测的统计模型。
