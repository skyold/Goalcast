## 📝 说明

### 为什么这样组织？

1. **清晰的文档结构**
   - 所有文档集中在 `docs/` 目录
   - 便于查找和维护
   - 保持根目录简洁

2. **版本控制友好**
   - 文档变更历史清晰
   - 不影响根目录的其他文件

3. **便于扩展**
   - 未来可以创建子目录分类
   - 例如：`docs/api/`, `docs/tutorials/` 等
